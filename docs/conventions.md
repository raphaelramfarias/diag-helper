# Padrões e Convenções

Este documento define os padrões adotados no projeto para manter
consistência e qualidade no código.

## Nomenclatura

- Componentes: PascalCase  
  Exemplo: `PatientForm.jsx`
- Funções e variáveis: camelCase  
  Exemplo: `handleSubmit`
- Pastas: lowercase  
  Exemplo: `components`, `pages`

## Organização

- Um componente por arquivo
- Componentes reutilizáveis ficam em `components`
- Páginas completas ficam em `pages`

## Estilo de código

- ESLint configurado para manter padrão
- Evitar código duplicado
- Funções pequenas e objetivas

## Boas práticas

- Comentários apenas quando necessário
- Código deve ser autoexplicativo
- Separação clara de responsabilidades
